57  /*!
58  * \brief
59  * Painter implements a canvas rendering interface.
60  *
61  * Painter implements:
62  * - stroking
63  * - filling
64  * - drawing text
65  * - applying a brush (see PainterBrush)
66  * - single 3x3 transformation
67  * - save and restore state
68  * - clipIn against Path, rectangle or rounded rectangle
69  * - clipOut against Path, rectangle or rounded rectangle
70  *
71  * The transformation of a Painter goes from local item coordinate
72  * to 3D API clip-coordinates (for example in GL, from item coordinates
73  * to gl_Position.xyw). FastUIDraw follows the convention that the top
74  * of the window is at normalized y-coordinate -1 and the bottom of the
75  * window is at normalized y-coordinate +1. The transformation is to be
76  * applied as matrix-vector multiplication, i.e.
77  * \code
78  * ClipCoordinates = transformation().m_item_matrix * vec3(x, y, 1.0)
79  * \endcode
80  * for local coordiante (x, y). Normalized device coordinates are
81  * defined as
82  * \code
83  * NormalizedDeviceCoordinates = ClipCoordinates.xy / ClipCoordinates.w
84  * \endcode
85  * where (-1, -1) corresponds to the bottom-left hand corner of the
86  * viewport (see PainterBackend::Surface::viewport()) and (+1, +1)
87  * is the top right hand corner of the viewport.
88  */
