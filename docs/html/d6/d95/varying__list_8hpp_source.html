33  /*!
34  * \brief
35  * A varying_list lists all the in's of a frag shader (and
36  * their names) which is the same as the out's of the vertex
37  * shader which which it is paired.
38  *
39  * A varying is ALWAYS a SCALAR. The varyings of shaders should
40  * -never- be declared in the shader code. Instead, each varying
41  * should be declared in the \ref varying_list object passed to
42  * the shader object's ctor. The \ref GLSL module will share the
43  * varyings across different shaders within the uber-shader.
44  * Indeed, the number of varying the uber-shader has is not the
45  * sum of the varyings across all the shaders, but rather is the
46  * maximum number of varyings across the shaders present in the
47  * uber-shader.
48  */
49  class varying_list
50  {
51  public:
